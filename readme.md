# Подпись PDF (ГОСТ-ЭП)

**https://agran.github.io/pdf-gost-signer/**

Веб-приложение для подписания PDF электронной подписью **ГОСТ Р 34.10-2012** полностью в браузере — без отправки документа на сервер.

Использует плагин КриптоПро ЭЦП (CryptoPro CAdES Browser Plug-in). Подпись выполняется по стандарту **PAdES-BES** (ETSI EN 319 142-1) методом инкрементального обновления.

На первой странице PDF размещается видимая печать по **ГОСТ Р 7.0.97-2016**: логотип, заголовок «ДОКУМЕНТ ПОДПИСАН / ЭЛЕКТРОННОЙ ПОДПИСЬЮ», серийный номер сертификата, владелец и срок действия.

---

## Возможности

- Подпись PDF (PAdES-BES detached)
- Видимая печать на первой странице (логотип + реквизиты сертификата)
- Выбор сертификата из хранилища КриптоПро
- Предпросмотр PDF с drag-and-drop позиционированием печати
- Скачивание подписанного PDF или ZIP-архива (PDF + .sig)
- Всё в браузере — документ не покидает компьютер

## Требования

- КриптоПро CSP (≥ 4.0)
- [CryptoPro Extension for CAdES Browser Plug-in](https://docs.cryptopro.ru/cades/plugin/plugin-installation-windows)
- Файл `cadesplugin_api.js` в той же директории
- HTTPS или `localhost` (HTTP не работает для доступа к плагину)
- Домен добавлен в разрешения расширения

---

## Как это работает

Подпись встраивается методом **инкрементального обновления** — оригинальные байты PDF не изменяются. В конец файла дописываются:

```text
[Оригинальный PDF]
[Новые объекты]      — AcroForm, SigField, SigDict, AppXObject, Catalog, Page
[Новая xref-таблица] — только новые объекты
[Новый trailer]      — /Prev → старый xref
```

**Плейсхолдер подписи** — 200 000 hex-символов (100 КБ) в `/Contents` словаря подписи, заполненных `0`. После получения подписи она вставляется в hex-формате, оставшееся место добивается нулями. Длина файла не меняется — это критично для `/ByteRange`.

**Алгоритм подписания** (три стратегии):

1. `SignCades detached` с `CHAIN_EXCEPT_ROOT` — предпочтительный, компактная отделённая подпись
2. `SignCades attached` — запасной
3. `RawSignature.SignHash` — ручная подпись хеша

## Визуальная печать

Реализована как Form XObject на первой странице PDF. Шрифт — Type1 Helvetica с кастомной кириллической кодировкой через `/Encoding /Differences` с AFII-именами глифов (afii10017–afii10097).

Макет:

```text
┌─────────────────────────────────────────┐
│ [logo]  ДОКУМЕНТ ПОДПИСАН               │
│         ЭЛЕКТРОННОЙ ПОДПИСЬЮ            │
│                                         │
│ Сертификат: 00D6AF04844D...             │
│ Владелец: Иванов Иван Иванович          │
│ Действителен: с 31.03.23 по 30.06.24    │
└─────────────────────────────────────────┘
```

- Рамка со скруглёнными углами (радиус 8 pt, кривые Безье)
- Размеры по умолчанию: 204×78 pt
- Позиция печати выбирается drag-and-drop в превью
- Логотип — JPEG (`/DCTDecode`), определяется автоматически по домену или выбирается вручную

## Ключевые константы

| Константа         | Значение                | Назначение                  |
|-------------------|-------------------------|-----------------------------|
| `RESERVE_SIZE_HEX`| 200000 (hex-символов)   | Плейсхолдер `/Contents`     |
| `/SubFilter`      | `/ETSI.CAdES.detached`  | Тип подписи по ETSI         |
| `/Filter`         | `/Adobe.PPKLite`        | Стандартное значение        |
| Ширина печати     | 204 pt                  | `Math.min(pw - 80, 204)`    |
| Высота печати     | 78 pt                   | Фиксировано                 |
| Радиус углов      | 8 pt                    | Кривые Безье                |

---

## Ограничения

- Размер подписи не более 100 КБ (при превышении нужно увеличить `RESERVE_SIZE_HEX`)
- Логотип только JPEG
- Проверка подписи в Adobe Reader требует установленного КриптоПро CSP
- В ZIP-версии (PDF + .sig) нет визуальной печати — только в подписанном PDF
